# Prettier reformats generated files after commit, breaking byte-equality drift checks

## Problem

Generated files committed for CI byte-equality checking (e.g., `.github/copilot-instructions.md` produced by `scripts/build-copilot-instructions.ts`) are silently reformatted by the `lint-staged` pre-commit Prettier hook. The next time the generator runs, its output differs from the committed bytes and the CI `--check` step fails on a file the developer never modified by hand.

## Symptoms

- `.github/copilot-instructions.md` matched `npm run build:copilot-instructions:check` before staging but diverged after `git commit`
- Prettier padded markdown table columns from `| --- | --- |` to `| ------- | ------- |`
- Prettier replaced `*asterisk*` italic delimiters with `_underscore_`
- CI's "Verify .github/copilot-instructions.md is current" step would have failed on the first run after merge

## Root Cause

Prettier's `lint-staged` hook runs after `git add` but before the commit is finalized. Even if `npm run build:foo:check` passes when manually verified pre-commit, the bytes that actually land in HEAD have been touched by the formatter between staging and the commit landing.

## Solution

Add the generated file to `.prettierignore` with a comment explaining why:

```
package-lock.json
# Generated by scripts/build-copilot-instructions.ts; CI verifies byte-for-byte
# via `npm run build:copilot-instructions:check`. Prettier reformatting would
# silently drift the file from what the script produces.
.github/copilot-instructions.md
```

Then regenerate (`npm run build:copilot-instructions`) and commit. The bytes now stay stable across commit cycles.

## Prevention

Any tracked, generated artifact whose CI gate is byte-equality with the generator's output MUST be in every formatter's ignore list — Prettier, ESLint `--fix`, dprint, biome, gofmt, etc. The pattern applies equally to generated TypeScript stubs, generated JSON schemas, generated Markdown docs, and generated workflow YAML.

To detect at PR time, run `npm run build:foo:check` AFTER the commit lands (not before staging), since the formatter mutation happens inside the commit boundary.
