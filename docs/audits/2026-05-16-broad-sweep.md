## Post-Audit Notes

- Baseline lint warnings at audit start: ProductChip missing `translateY` hook dependency; CoachHint-utils unused `ScanPhase`; useCameraPermissions unused `useState`; TastePicksGrid unused `withOpacity`; CoachChat unused `CoachStatusRow` and unnecessary callback deps; CuratedRecipeCarousel unused `CARD_WIDTH`; ScanScreen unused `ContentType`; evals judge unused `judgeGeneric`.
- Discovery agents: security, performance, data-integrity, architecture, camera, and accessibility reported findings; code-quality returned no reportable output. Camera/accessibility duplicate modal-focus findings were merged into H4.
- Triage pass: keep H1-H4, M4, M5, M8, M9, M10, L2, and L4 open for the first fix loop; defer 14 findings into 7 linked todos.
- H2 fixed first because it was a contained data-integrity correctness bug with existing storage test coverage.
- H1 fixed second. Kimi review's sanitizer warning conflicted with the explicit all-roles sanitization rule and was declined; its separate verification aggregate concurrency warning was deferred as a new todo.
- M5 fixed third, reusing the same hidden-target-is-not-orphan rule codified from the cookbook audit.
- H3 fixed fourth. Streaming coach and recipe assistant UI now renders from memoized `ListFooterComponent` footers instead of high-frequency sentinel rows in `FlatList.data`.
- H4 fixed fifth. RootStack modal/fullScreenModal screens now set `accessibilityViewIsModal` on their scene roots; Kimi's non-modal criticism was adjudicated as a false-positive against the binding accessibility rule.
- M4 fixed sixth. Daily logs now match daily summary behavior by excluding logs attached to discarded scanned items.
- M8 fixed seventh. Post-response recipe image generation now uses labeled `fireAndForget`; reviewer concerns about the helper lacking `.catch()` were verified false against the helper implementation.
- L2 fixed eighth. The micronutrient reference endpoint now uses the same rate limiter as sibling micronutrient endpoints.
- L4 fixed ninth. Adaptive-goals settings now validates the request body before field access.
- M9 fixed tenth. Photo analysis cancellation now owns and aborts the current controller on effect cleanup, passes the signal into the upload helper, and skips state/haptic updates after abort; native upload cancellation remains limited by Expo `uploadAsync`.
- M10 fixed eleventh. Receipt and cook compressed upload files now clean up from `finally` blocks so failures do not leak temp images.
