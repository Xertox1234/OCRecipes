# React Native Rules

- Never pass functions or callbacks as route params — use event params or navigate + read-and-clear in `useEffect` (non-serializable params break state persistence and deep links)
- Always call `navigation.goBack()` immediately after `navigation.navigate()` when dismissing a `fullScreenModal` — `navigate()` alone leaves the modal on the stack
- Touch targets must be ≥ 44pt — add `hitSlop` for small controls (e.g., `hitSlop={{ top: 8, bottom: 8, left: 8, right: 8 }}`)
- Always add `insets.bottom + Spacing.X` to the last-item padding in scrollable lists — bare `Spacing.X` clips content behind the home indicator on notched devices
- `KeyboardAvoidingView` behavior: `"padding"` on iOS, `"height"` on Android — never `undefined`
- Always spread `FLATLIST_DEFAULTS` from `@/constants/performance` on every `FlatList` component
- Camera: always set `isActive={isFocused}` on `CameraView` — stops the camera when navigating away
- Mic: always call `stopListening()` in session reset/cleanup — leaving it open mutates state after the component is gone
- `Alert.prompt` is iOS-only — always guard with `Platform.OS === "ios"` and provide a `TextInput` fallback for Android
- Never gate `Linking.openURL(httpUrl)` behind `Linking.canOpenURL` — `canOpenURL` returns false on iOS for HTTP(S) without `LSApplicationQueriesSchemes` entries and blocks legitimate links. Call `openURL` directly and `try/catch` the failure instead
- A deep-link target screen must distinguish an **omitted** id param (the in-app create flow) from a **present-but-invalid** one. `linking.ts` `parseIntOrZero` coerces a malformed path segment to `0`, so deriving create-mode from a falsy check (`!entryId`) treats a bad link as "create" → it silently opens a blank create form and a save spawns a duplicate. Derive create-mode from `param === undefined`, and for a present id that doesn't resolve render a **load-aware** not-found state (gate on the list query's `isLoading`/`isPending`, not just `!entry`, or it flashes not-found while loading). Precedent: `client/screens/NotebookEntryScreen.tsx` (H4, 2026-05-29 reliability audit).
- A `beforeRemove` discard guard fires for the screen's OWN forward navigation too (`replace` removes the screen) — whitelist each intentional removal by `e.data.action.type` + payload target before `preventDefault()`, or the primary flow shows the discard confirm and can wipe state the destination reads. See `docs/solutions/logic-errors/before-remove-guard-intercepts-own-replace-2026-06-10.md`
- A function imported from another module and called inside a Reanimated worklet body (`runOnUI`, `useAnimatedStyle`, `useAnimatedScrollHandler`, a Gesture builder callback, etc.) needs its own `"worklet"` directive as the first statement — the Babel plugin does not workletize across `import` edges, so a missing directive is a silent app close in release/OTA builds despite `tsc`/ESLint/unit tests all passing. Enforced by the static guard in `scripts/__tests__/worklet-directive-guard.test.ts`. See `docs/solutions/runtime-errors/reanimated-worklet-util-needs-directive-across-imports-2026-06-27.md`
