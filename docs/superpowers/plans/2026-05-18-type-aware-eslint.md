# Type-Aware ESLint Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add `@typescript-eslint` type-checked rules to catch unhandled async, enforced in CI via a native ESLint bulk-suppressions ratchet so the existing backlog never blocks but new violations always fail.

**Architecture:** One conditional block is appended to the existing flat `eslint.config.js`. The block is omitted when `ESLINT_NO_TYPE_AWARE` is set, which keeps pre-commit fast. Today's violations are snapshotted into a committed `eslint-suppressions.json`; CI gates new violations and a staleness check keeps the snapshot pruned.

**Tech Stack:** ESLint 9 flat config, `typescript-eslint` v8 (type-checked rules + `projectService`), Husky + lint-staged, GitHub Actions.

**Note on verification:** This is tooling/config work. There are no unit tests — each task is verified by running a command and confirming its output. Treat the "verify" steps as the equivalent of test steps.

**Source spec:** `docs/superpowers/specs/2026-05-18-type-aware-eslint-design.md`

## Task 1: Add `typescript-eslint` and verify version alignment

`eslint-config-expo` already pulls `@typescript-eslint/*` packages in transitively. Adding the `typescript-eslint` v8 meta-package on top can create a version skew where the parser's AST shape does not match what the rules expect, silently no-op'ing rules. This task installs the package and forces a single resolved version.

**Files:**

- Modify: `package.json` (add devDependency, possibly an `overrides` block)
- Modify: `package-lock.json` (regenerated by npm)

- [ ] **Step 1: Install the meta-package**

Run:

```bash
npm install -D typescript-eslint
```

Expected: installs without peer-dependency errors. If npm prints `ERESOLVE` peer warnings about `@typescript-eslint/*`, note them — Step 3 resolves them.

- [ ] **Step 2: Check for version skew**

Run:

```bash
npm ls typescript-eslint @typescript-eslint/parser @typescript-eslint/eslint-plugin @typescript-eslint/utils @typescript-eslint/type-utils
```

Expected: each package name should resolve to **exactly one** version. A "deduped" line is fine. If any package shows **two different versions** (one under `typescript-eslint`, one under `eslint-config-expo`), there is a skew — proceed to Step 3. If every package shows a single version, **skip Step 3**.

- [ ] **Step 3 (conditional): Force version alignment with `overrides`**

Only if Step 2 showed a skew. Read the version that the `typescript-eslint` meta-package resolved (the version shown directly under `typescript-eslint` in the Step 2 output — call it `<V>`, e.g. `8.21.0`). Add an `overrides` block to `package.json` at the top level (sibling of `devDependencies`), substituting the real version for `<V>`:

```json
  "overrides": {
    "@typescript-eslint/parser": "<V>",
    "@typescript-eslint/eslint-plugin": "<V>",
    "@typescript-eslint/utils": "<V>",
    "@typescript-eslint/type-utils": "<V>"
  }
```

Then run:

```bash
npm install
npm ls @typescript-eslint/parser @typescript-eslint/eslint-plugin @typescript-eslint/utils @typescript-eslint/type-utils
```

Expected: each package now resolves to the single version `<V>`.

- [ ] **Step 4: Verify lint still passes (no behavior change yet)**

The dependency is installed but not referenced by any config, so linting is unchanged.

Run:

```bash
npm run lint
```

Expected: exits 0 (same as before this task).

- [ ] **Step 5: Commit**

```bash
git add package.json package-lock.json
git commit -m "build: add typescript-eslint for type-aware lint rules"
```

## Task 3: Add the type-aware config block, scripts, and suppressions snapshot

Done in one task so every commit leaves `npm run lint` green: the config block alone would fail lint (unsuppressed backlog), so the block, the scripts, and the snapshot land together.

**Files:**

- Modify: `eslint.config.js`
- Modify: `package.json` (add three scripts)
- Create: `eslint-suppressions.json` (generated, committed)

- [ ] **Step 1: Add the `typescript-eslint` require**

In `eslint.config.js`, add this line after the existing `require` lines at the top of the file (after the `ocrecipesPlugin` require):

```js
const tseslint = require("typescript-eslint");
```

- [ ] **Step 2: Append the type-aware block**

In `eslint.config.js`, insert this block inside the `defineConfig([...])` array, immediately before the closing `]);`:

```js
  // Type-aware async-safety rules. Omitted when ESLINT_NO_TYPE_AWARE is set
  // (pre-commit), so commits stay fast. CI runs npm run lint without the flag.
  ...(process.env.ESLINT_NO_TYPE_AWARE
    ? []
    : [
        {
          files: ["**/*.{ts,tsx}"],
          languageOptions: {
            parser: tseslint.parser,
            parserOptions: {
              projectService: true,
              tsconfigRootDir: __dirname,
            },
          },
          plugins: { "@typescript-eslint": tseslint.plugin },
          rules: {
            "@typescript-eslint/no-floating-promises": "error",
            "@typescript-eslint/no-misused-promises": "error",
            "@typescript-eslint/await-thenable": "error",
            "@typescript-eslint/require-await": "error",
            "@typescript-eslint/no-implied-eval": "error",
            "@typescript-eslint/prefer-promise-reject-errors": "error",
          },
        },
      ]),
```

- [ ] **Step 3 (conditional): Apply Finding D1 — relax `checksVoidReturn`**

Only if Task 2 Finding D1 decided to relax it. In the block from Step 2, replace the `no-misused-promises` line with:

```js
            "@typescript-eslint/no-misused-promises": [
              "error",
              { checksVoidReturn: { attributes: false } },
            ],
```

If Finding D1 kept the default, leave the line as written in Step 2 and skip this step.

- [ ] **Step 4 (conditional): Apply Finding D2 — scope `require-await` off for tests**

Only if Task 2 Finding D2 decided to scope it off. In `eslint.config.js`, add this block immediately after the type-aware block from Step 2 (still inside the array, before `]);`):

```js
  ...(process.env.ESLINT_NO_TYPE_AWARE
    ? []
    : [
        {
          files: ["**/*.test.{ts,tsx}"],
          rules: { "@typescript-eslint/require-await": "off" },
        },
      ]),
```

If Finding D2 kept it global, skip this step.

- [ ] **Step 5: Verify the rule fires (deterministic check)**

Create a throwaway file inside the repo so `projectService` resolves it via `tsconfig.json`:

```bash
cat > server/__type_aware_lint_check.ts <<'EOF'
async function check() {
  return 1;
}
check();
EOF
npx eslint server/__type_aware_lint_check.ts
```

Expected: ESLint reports `@typescript-eslint/no-floating-promises` on the `check();` line. This proves the parser, `projectService`, and the rule are all wired.

Clean up:

```bash
rm server/__type_aware_lint_check.ts
```

- [ ] **Step 6: Add the three npm scripts**

In `package.json`, add these to the `"scripts"` object:

```json
    "lint:suppress": "eslint . --suppress-rule @typescript-eslint/no-floating-promises --suppress-rule @typescript-eslint/no-misused-promises --suppress-rule @typescript-eslint/await-thenable --suppress-rule @typescript-eslint/require-await --suppress-rule @typescript-eslint/no-implied-eval --suppress-rule @typescript-eslint/prefer-promise-reject-errors",
    "lint:suppress:prune": "eslint . --prune-suppressions",
    "lint:suppress:check": "eslint . --prune-suppressions && git diff --exit-code -- eslint-suppressions.json"
```

`lint:suppress` snapshots only the six type-aware rules into `eslint-suppressions.json` (using `--suppress-rule`, not `--suppress-all`, so an unrelated future rule is never over-suppressed). `lint:suppress:check` is the CI staleness gate from Task 5.

- [ ] **Step 7: Generate the suppressions snapshot**

Run:

```bash
npm run lint:suppress
```

Expected: creates `eslint-suppressions.json` in the repo root containing per-file, per-rule counts for the six rules.

- [ ] **Step 8: Confirm the snapshot file is not gitignored**

Run:

```bash
git check-ignore eslint-suppressions.json; echo "exit: $?"
```

Expected: `exit: 1` (not ignored — the file prints nothing and `git check-ignore` exits 1 when the path is not ignored). If it prints the filename with `exit: 0`, the file is ignored — remove the matching pattern from `.gitignore` before continuing, since the ratchet state must be committed.

- [ ] **Step 9: Verify lint now passes with the backlog suppressed**

Run:

```bash
npm run lint
```

Expected: exits 0. The backlog is absorbed by `eslint-suppressions.json`; only _new_ violations would fail.

- [ ] **Step 10: Commit**

```bash
git add eslint.config.js package.json eslint-suppressions.json
git commit -m "feat: enable type-aware ESLint rules with bulk-suppression ratchet"
```

## Task 4: Keep pre-commit fast (exclude type-aware rules from lint-staged)

`lint-staged` runs `eslint --fix` per staged file. Without this change, every commit would cold-start a TypeScript program. Setting `ESLINT_NO_TYPE_AWARE=1` makes the config omit the type-aware block during pre-commit.

**Files:**

- Modify: `package.json` (the `lint-staged` block)

- [ ] **Step 1: Prefix the lint-staged ESLint command**

In `package.json`, in the `"lint-staged"` block, change the `"*.{ts,tsx}"` entry's first command from:

```json
    "*.{ts,tsx}": [
      "eslint --fix",
      "prettier --write"
    ],
```

to:

```json
    "*.{ts,tsx}": [
      "ESLINT_NO_TYPE_AWARE=1 eslint --fix",
      "prettier --write"
    ],
```

The bare env-var prefix is POSIX shell syntax — fine for macOS dev and Ubuntu CI; the project has no Windows contributors.

- [ ] **Step 2: Verify the env var disables the type-aware block**

`--print-config` shows the resolved config for a file. With the flag set, the type-aware rules must be absent.

Run:

```bash
ESLINT_NO_TYPE_AWARE=1 npx eslint --print-config server/index.ts | grep -c 'no-floating-promises'
npx eslint --print-config server/index.ts | grep -c 'no-floating-promises'
```

Expected: first command prints `0` (rule absent — block omitted); second prints `1` or more (rule present — normal config). If `server/index.ts` does not exist, substitute any committed server `.ts` file.

- [ ] **Step 3: Commit**

```bash
git add package.json
git commit -m "build: exclude type-aware ESLint rules from pre-commit hook"
```

## Task 5: Add the CI staleness gate

CI's existing `npm run lint` step already gates new violations (it loads `eslint-suppressions.json` automatically). This task adds a second step that fails if the committed suppressions file contains stale entries — keeping the ratchet tight, mirroring the existing `build:copilot-instructions:check` pattern.

**Files:**

- Modify: `.github/workflows/ci.yml`

- [ ] **Step 1: Add the staleness-check step**

In `.github/workflows/ci.yml`, in the `checks` job, add this step immediately after the existing `Lint` step (after lines 33–34):

```yaml
- name: ESLint suppressions are pruned
  run: npm run lint:suppress:check
```

Placing it after `Lint` means it only runs when linting passed, so `eslint --prune-suppressions` exits 0 and the `git diff --exit-code` reliably detects a stale (un-pruned) `eslint-suppressions.json`.

- [ ] **Step 2: Verify the check passes locally**

Run:

```bash
npm run lint:suppress:check
echo "exit: $?"
```

Expected: `exit: 0`. The snapshot was generated fresh in Task 3, so pruning changes nothing and `git diff --exit-code` finds no diff.

- [ ] **Step 3: Commit**

```bash
git add .github/workflows/ci.yml
git commit -m "ci: gate on pruned ESLint suppressions file"
```
